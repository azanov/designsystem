Supported Animations

This library handles different types of animations. At the moment, this library is only compatable with Angular.JS. In order to use this library, make sure you have the ng-animate module included in your app.

Queue Based Animations

Focus Grabbing Animations

Most people will be primarily focussed on queue based animations. These animations are used for entrances and exits, and can do a great deal to smooth out the overall fluidity of an application.

Less commonly used are focus grabbing animations. These come into play when you think a user might be stuck on what step comes next.

Any animations applied should not interfere with a current animation that is running on an object. This means that these animations will only take place if angular events are fired. These events will work with either ng-if, ng-show or ng-hide. There are no restrictions on how these data attributes are added. If you would prefer to prefix your attributes with data- that will not interfere with the animations.

The pb-animate classes can be applied in two ways. You can quickly develop your code by adding classes directly to the HTML class attribute. However, for the sake of readability and template management, it is recommended that once you are satisfied with your final product that you move your classes into an @extend in Sass.
