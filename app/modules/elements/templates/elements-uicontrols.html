Input Fields

States

Input fields have four primary states, each defined by its form and type coloring:

  • Default
  • Active
  • Disabled
  • Error

Sizes

We have three sizes of input fields: small, default and large. Use the default size for all fields except when:

  • You have a very crowded form (20+ fields), in which case consider using small input fields.
  • You have only a few fields (<10), in which case consider using large input fields. An example of this is a Sign In screen.

Once you’ve selected an input field size, use it consistently throughout your product. The exceptions to this rule are pages that are outside of your product’s flow (such as Sign In or Sign Up), which should use the large size.

Note that even when using large or small input fields the label font remains the default size.

Colors

We apply red to field elements to alert users to an input error that needs their attention

Help Prompt

There are several ways to help users understand what they’re being asked to input. These include:

  • In-field prompt copy.
  • A short line of instructional copy below the field.
  • A small popover next to the field’s label that provide additional context.

Input groups

You can augment regular input field behavior by prepending or appending icons, checkboxes, buttons or dropdown menus to them.

Writing Field Copy

When writing your labels and in-field prompt copy, it’s important to stay consistent across your entire product. Here are some basic guidelines for how to write these critical pieces of user interface copy.

Style

  • Use short, active words and phrases.
  • Structure copy to avoid ambiguity in language.
  • To avoid confusing users, use consistent terminology.

Capitalization

  • Use sentence capitalization, except when copy includes words or phrases that are acronyms or proper nouns (e.g., “RMA number,” “Cost Account,” “ZIP Code”).

Punctuation

  • Don’t conclude copy with punctuation, such as periods or colons.
  • Don’t phrase your copy as questions.

Dropdown Menus

Usage

  • Best suited for lists containing 7-15 items. For fewer items items consider using radio buttons.
  • Never present the user with a blank menu. In most cases, “Select” should be your first menu item, to help prompt the user to make a selection.
  • Use extreme caution with preselecting an item, especially in required or critical menus — only do so if at least 80% of users are likely to select that item.

States

Dropdown menus have four primary states, each defined by its form and type coloring:

  • Default
  • Focused
  • Active
  • Disabled

Sizes

We have three sizes of dropdown menus: default, large and small. Use the default size for all dropdowns except when:

  • You have a very crowded form (20+ dropdowns), in which case consider using small dropdown menus.
  • You have only a few dropdowns (<10), in which case consider using large dropdown menus.

Once you’ve selected a dropdown menu size, use it consistently throughout your product. The exceptions to this rule are pages that are outside of your product’s flow (such as Sign Up), which should use the large size.

Note that even when using large or small dropdown menus the label font remains the standard size.

Multiple Selection

When using multiple selection menus, consider letting users know which keys they need to use (Ctrl-click or Cmd-click) in order to use this feature. Or consider using the multi-select with tokens menu.

Advanced Select Menus

You can use these advanced menu types to augment our standard dropdown menus. “Multi-select with tokens” is a good alternative to the multi-section types above, as it allows users to create a persistent list of selections that they can easily edit.

Writing Dropdown Copy

When writing labels and menu items for dropdowns, it’s important to stay consistent across your entire product. Here are some basic guidelines for how to write these critical pieces of user interface copy.

Style

  • Use short, active words and phrases.
  • Structure copy to avoid ambiguity in language.
  • To avoid confusing users, use consistent terminology.

Capitalization

  • Labels use sentence capitalization, except when copy includes words or phrases that are acronyms or proper nouns, which should be capitalized.
  • Menu items use title capitalization, because they behave like buttons.

Punctuation

  • Don’t conclude copy with punctuation, such as periods or colons.
  • Don’t phrase your copy as questions.

Checkboxes

Styles

We have two styles of checkboxes: enhanced and standard OS.

Usage

  • When possible, you should use the enhanced style.
  • If your product has long lists of checkboxes (and space is an issue), consider using the standard OS.
  • Use the same style for your checkboxes and radio buttons, and apply that style consistently across your product.

Writing Checkbox Labels

When writing your labels, be descriptive enough so users understand what checking the box will do, but try to avoid very long labels. One way to do this is by introducing the checkboxes with a descriptive piece of copy that lends context to the checkboxes’ purpose.

Steppers

Numeric Stepper

Use the stepper when you want users to select incrementally from a wide range of values, or input their own value.

Steppers, like all Boostrap form controls, expand to fill the width of their container.
