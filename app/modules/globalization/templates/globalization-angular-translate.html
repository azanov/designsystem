Angular Translate

angular-translate is an AngularJS module that makes development much easier when it comes to i18n and l10n, including lazy loading. It provides components like filters and directives, asynchronous loading of i18n data and much more! angular-translate is very flexible. You can build your own loaders, storages or error handlers and extend angular-translate to your needs!

The angular-translate Getting Started guide provides detailed information on localizing you application.

You develop the application in a native language (English, for example) using standard JSON files with keys for each string. An sample of the JSON is shown after the example below. The JSON keys are used in views and controllers, instructing angular-translate which string to display. The Globalization team consumes the native language JSON files in an application called Passolo and provides JSON files for each translated language. As you update your native language file, Passolo is used to track the differences in the files, allowing translators to focus on what has changed, saving time and budget!

As a best practice, files should be named by langauge and country, all lowercase. For example, en-us.json, fr-fr.json, ja-jp.json or pt-br.json.

Contact the Globalization Team at globalization@pb.com or through the Yammer group for more information.

Angular i18n & Angular Dynamic Locale

Angular separates number, date and time, and currency formats into different files for each locale. Angular provides filters for date, time and currency that use the loacale information to display the content in the correct format. You can find a list of currently supported locales here and more information on Angular i18n/l10n here.

In order to change the locale dynamically in the view, include the angular-dynamic-locale library in your application. This library will allow you to set the locale based on a selection and store or retrieve the locale from a cookie.
